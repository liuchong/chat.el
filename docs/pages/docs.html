Documentation

Complete guide to installing, configuring, and using chat.el

Installation

chat.el is a pure Emacs Lisp package with no external dependencies beyond Emacs 27+.

Manual Installation

  1. Clone the repository:
git clone https://github.com/liuchong/chat.el.git
  1. Add to your Emacs configuration:
(add-to-list 'load-path "/path/to/chat.el")
(require 'chat)

Configuration

Provider Setup

Configure your preferred LLM provider:

Kimi Code

(setq chat-default-model 'kimi-code)
(setq chat-llm-kimi-code-api-key "sk-kimi-...")

Kimi

(setq chat-default-model 'kimi)
(setq chat-llm-kimi-api-key "sk-...")

OpenAI Compatible

(setq chat-default-model 'openai)
(setq chat-llm-openai-api-key "sk-...")
(setq chat-llm-openai-base-url "https://api.openai.com/v1")

Secure Credential Storage

Recommended: Use auth-source instead of hardcoding API keys:

# ~/.authinfo or ~/.authinfo.gpg
machine kimi-code-api user api-key password YOUR_KEY
machine openai-api user api-key password YOUR_KEY

Recommended Configuration

(setq chat-default-model 'kimi-code)
(setq chat-ui-use-streaming t)
(setq chat-session-auto-save t)
(setq chat-files-allowed-directories
      (list default-directory "/tmp/" "/var/tmp/"))

Quick Start

Start a chat session with:

M-x chat

This will open or resume the current chat buffer. Type your message and press C-c C-c to send.

Commands

Command Description
M-x chat Open or resume the current chat buffer
M-x chat-new-session Create a new session
M-x chat-list-sessions Switch to an existing session
M-x chat-view-raw-message Inspect the last raw API exchange
M-x chat-view-last-raw-exchange Open the latest assistant request and response
M-x chat-ui-cancel-response Cancel the active response

LLM Providers

chat.el supports multiple LLM providers through a unified abstraction layer.

Supported Providers

  • Kimi — Moonshot AI's general-purpose model
  • Kimi Code — Optimized for coding tasks with China region support
  • OpenAI Compatible — Any OpenAI-compatible API endpoint

Adding Custom Providers

Extend chat-llm.el to add new providers. See existing implementations in chat-llm-kimi.el and chat-llm-openai.el.

Tool System

chat.el includes a powerful tool calling system that allows the AI to interact with your filesystem and execute commands.

Built-in Tools

Tool Description Risk Level
files_read Read file contents Low
files_read_lines Read specific line ranges Low
files_list List directory contents Low
files_grep Search file contents Low
files_write Write file contents High
files_replace Replace file contents High
files_patch Apply patches to files High
apply_patch Apply unified diff patches High

Approval System

High-risk tools require explicit user approval before execution. The approval flow:

  1. AI requests to use a risky tool
  2. User reviews the proposed action
  3. User approves or denies
  4. Result is fed back to the AI

AI-Generated Tools

chat.el can generate custom tools through AI assistance. Generated tools:

  • Must be a single top-level lambda form
  • Require approval before registration
  • Are saved to disk for persistence
  • Are compiled and loaded dynamically

Session Management

Sessions are automatically saved to disk, allowing you to resume conversations across Emacs restarts.

Session Features

  • Auto-save — Sessions persist automatically
  • Context trimming — Long conversations are intelligently trimmed
  • System message preservation — Important context is retained
  • Summary generation — Conversation summaries for long threads

Session Commands

M-x chat-new-session        ; Create new session
M-x chat-list-sessions      ; List and switch sessions
M-x chat-view-raw-message   ; Inspect API exchanges

Architecture

chat.el follows a modular architecture with clear separation of concerns:

Module Responsibility
chat.el Entry point and command wiring
chat-ui.el Chat buffer rendering and response lifecycle
chat-session.el Session and message persistence
chat-llm.el Provider abstraction and async request handling
chat-stream.el SSE parsing and chunk handling
chat-tool-caller.el Tool prompt contract, parsing, and execution
chat-approval.el Approval flow for risky tools
chat-files.el Built-in file tools and path safety
chat-context.el Context trimming and summary generation
chat-tool-forge.el Tool registry, compilation, loading
chat-tool-forge-ai.el AI-assisted tool generation

File Access

By default, file tools can only access:

  • The current project directory
  • /tmp/
  • /var/tmp/

Override with:

(setq chat-files-allowed-directories
      (list default-directory "/tmp/" "/var/tmp/" "/custom/path/"))

Troubleshooting

Common Issues

API Key Not Found

Ensure your API key is set or use auth-source:

(setq chat-llm-kimi-code-api-key "your-key")
;; or
(machine kimi-code-api user api-key password YOUR_KEY)

File Access Denied

Check chat-files-allowed-directories includes your target path.

Streaming Not Working

Ensure chat-ui-use-streaming is enabled and your provider supports SSE.
